为什么需要哈希表？

传统数组查找的瓶颈

在数组中查找特定元素（如学生姓名）需要遍历，时间复杂度为 O(n)，数据量大时效率低下。

for(int i = 0; i < 1000; i++) {
    if(students[i] == "李四") {
        cout << scores[i];
        break;
    }
}

哈希核心思想：键值对映射

通过哈希函数将“键”（如姓名）直接映射到存储位置，实现接近 O(1) 的查找速度。

理想用法：hashTable["李四"] → 直接获取成绩

哈希表数据结构设计

节点结构体

每个节点存储键、值和指向下一个节点的指针（用于处理冲突）。

struct StudentNode {
    string name; // 键 (Key)
    int score; // 值 (Value)
    StudentNode* next; // 链表指针
};

哈希表全局变量

使用指针数组作为哈希表的底层存储。

const int TABLE_SIZE = 20;
StudentNode* hashTable[TABLE_SIZE]; // 指针数组

哈希函数设计

将字符串键转换为数组索引。

int hashFunction(string name) {
    int sum = 0;
    for(int i = 0; i < name.length(); i++) {
        sum += (int)name[i]; // 累加ASCII码
    }
    return sum % TABLE_SIZE; // 取模确保在范围内
}

哈希表核心操作

初始化

void initHashTable() {
    for(int i = 0; i < TABLE_SIZE; i++) {
        hashTable[i] = nullptr;
    }
    cout << "哈希表初始化完成！大小：" << TABLE_SIZE << endl;
}

插入

void insertStudent(string name, int score) {
    int index = hashFunction(name);
    StudentNode* newNode = new StudentNode;
    newNode->name = name;
    newNode->score = score;
    newNode->next = hashTable[index];
    hashTable[index] = newNode;
}

查找

int searchStudent(string name) {
    int index = hashFunction(name);
    StudentNode* current = hashTable[index];
    while(current != nullptr) {
        if(current->name == name) {
            return current->score;
        }
        current = current->next;
    }
    return -1; // 未找到
}

哈希冲突与解决

什么是哈希冲突？

不同的键通过哈希函数计算后得到相同的索引值。

例如：hash("abc") = (97+98+99)%20 = 14

例如：hash("cba") = (99+98+97)%20 = 14

两者都映射到索引 14！

解决方法：链地址法

在每个哈希表索引位置维护一个链表，所有冲突的元素都链接在该链表上。

查找时，先通过哈希函数定位到链表头，再遍历链表进行匹配。

🎯 完整程序与测试

完整程序框架

#include <iostream>
#include <string>
using namespace std;

// 数据结构定义
struct StudentNode { ... };
const int TABLE_SIZE = 20;
StudentNode* hashTable[TABLE_SIZE];

// 函数声明
void initHashTable();
int hashFunction(string name);
void insertStudent(string name, int score);
int searchStudent(string name);
void displayHashTable();
void cleanupHashTable(); // 内存清理

int main() {
    initHashTable();
    insertStudent("张三", 95);
    insertStudent("李四", 88);
    insertStudent("王五", 92);
    searchStudent("张三");
    displayHashTable();
    cleanupHashTable();
    return 0;
}

内存管理：防止泄漏

使用 new 分配的内存必须手动释放。

void cleanupHashTable() {
    for(int i = 0; i < TABLE_SIZE; i++) {
        StudentNode* current = hashTable[i];
        while(current != nullptr) {
            StudentNode* temp = current;
            current = current->next;
            delete temp;
        }
        hashTable[i] = nullptr;
    }
    cout << "哈希表清理完成！" << endl;
}

性能对比

查找方式 平均时间复杂度 最坏情况
数组遍历 O(n) O(n)
哈希表 O(1) O(n)

性能关键：哈希函数质量、表大小（负载因子）、冲突解决策略。
