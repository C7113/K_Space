为什么需要格式化 I/O？

格式化的作用

格式化输入输出（I/O）是程序与用户交互的重要方式。它允许我们：

  • 精确控制数据格式：对齐、宽度、小数位数等
  • 提高可读性：整洁一致的输出更易于阅读
  • 方便数据解析：从日志、配置文件中读取固定格式数据
  • 国际化支持：按地区习惯显示数字、日期和时间
  • 保证兼容性：满足老系统或设备的特定格式要求

格式化输入

使用 cin

#include <iostream>
using namespace std;
int main() {
    int num;
    cin >> num;
    return 0;
}

使用 scanf

#include <stdio.h>
int main() {
    int num;
    scanf("%d", &num);
    return 0;
}

scanf 格式符

格式符 说明
d, i 输入十进制整数
u 无符号十进制整数
o 八进制整数
x, X 十六进制整数
c 输入单个字符
s 输入字符串（非空格开始，空格结束）
f, e 输入实数（小数或指数）

scanf 附加格式说明符

附加格式 说明
l (字母) 长整型(%ld)或double(%lf)
h 短整型(%hd)
域宽（整数） 指定输入列宽，如%5d
* 跳过输入，不赋给变量

格式化输出

使用 cout

cout << "结果：" << num;

使用 printf

printf("结果：%d", num);

printf 格式符

格式符 说明
d, i 带符号十进制整数
u 无符号十进制整数
o 八进制整数
x, X 十六进制整数
c 输出单个字符
s 输出字符串
f 小数形式输出浮点数（默认6位小数）
e, E 指数形式输出浮点数
g, G 自动选择 %f, %e, %E 中较短的格式

printf 高级格式控制

格式 说明
%d 按实际长度输出
%md 输出m位，不足补空格
%-md 左对齐输出
%ld 输出长整型
%mld 指定长整型输出宽度
%0md 不足m位时前面补0
%.mlf 保留m位小数
%.*lf 动态指定小数位数，如 printf("%. * lf", n, x)

🎯 项目实战

项目1：空格分隔输出

输入字符、整数、单精度、双精度浮点数，用空格分隔输出，浮点数保留6位小数。

char c; int i; float f; double d;
cin >> c >> i >> f >> d;
printf("%c %d %.6f %.6f", c, i, f, d);

项目2：电子表格式

将秒数转换为 xx:xx:xx 格式。

int total, h, m, s;
cin >> total;
h = total / 3600;
m = (total % 3600) / 60;
s = total % 60;
printf("%02d:%02d:%02d", h, m, s);

项目3：时间差

计算两个时间点之间的秒数差。

int h1, m1, s1, h2, m2, s2;
scanf("%d:%d:%d", &h1, &m1, &s1);
scanf("%d:%d:%d", &h2, &m2, &s2);
int t1 = h1*3600 + m1*60 + s1;
int t2 = h2*3600 + m2*60 + s2;
cout << t2 - t1;

项目4：竖式计算

输出 18+870 的竖式。

cout << " 18" << endl;
cout << "+ 870" << endl;
cout << "--------" << endl;
cout << " 888" << endl;

✅ 优点与 ❌ 弊端

优点

  • 精确控制格式：可设置对齐、宽度、填充等
  • 易于阅读：整齐的输出提升可读性
  • 数据解析方便：适用于固定格式的数据源
  • 国际化支持：适配不同地区的显示习惯
  • 性能优化：在某些场景下减少处理开销
  • 兼容性好：满足特定系统或设备的格式要求

弊端

  • 复杂性高：需要熟悉复杂的格式规则
  • 易出错：格式符与类型不匹配会导致错误
  • 可维护性差：格式需求变化时修改困难
  • 性能开销：处理大量数据时可能增加开销
  • 可移植性问题：不同平台/语言格式可能不同
  • 灵活性降低：格式固定，难以动态调整
