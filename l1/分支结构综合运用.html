🔍 核心知识点详解（查漏补缺）

1. 分支结构类型对比

类型 语法示例 适用场景
单分支 if (condition) { ... } 仅当条件成立时执行操作
双分支 if (cond) { ... } else { ... } 二选一逻辑（是/否）
多重分支 if...else if...else 或 switch 多条件互斥判断

2. switch 语句注意事项（补充原文件未详述内容）

  • 仅支持整型、字符型或枚举类型（不支持浮点数或字符串）
  • 每个 case 后建议加 break，否则会发生“穿透”（fall-through）
  • 必须包含 default 分支以处理异常输入，提高程序健壮性
switch (grade) { case 'A': cout << "优秀"; break; case 'B': cout << "良好"; break; default: cout << "成绩无效"; break; }

3. 逻辑运算符与短路机制（重要补充）

  • &&（与）：前为假则不计算后
  • ||（或）：前为真则不计算后

✅ 利用短路机制可避免运行时错误：

// 安全访问指针 if (ptr != nullptr && ptr->isValid()) { ... } // 防止除零 if (denominator != 0 && numerator / denominator > 5) { ... }

4. 嵌套优化技巧（提升代码质量）

避免深层嵌套（“箭头代码”），推荐使用“提前返回”（Guard Clause）：

// 不推荐 if (user) { if (user->active) { if (user->hasPermission) { doAction(); } } } // 推荐 if (!user) return; if (!user->active) return; if (!user->hasPermission) return; doAction();
✅ 减少嵌套可显著提升代码可读性和维护性！

5. 常见陷阱与调试建议

  • 混淆赋值（=）与比较（==）：编译器警告可帮助发现
  • 浮点数比较应使用误差范围：abs(a - b) < 1e-6
  • 多条件组合时注意括号明确优先级：if ((a || b) && c)
